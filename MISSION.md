# Mission

Learn React well enough to get and hold a **job** as a React developer.

## Why
The user is pursuing React for career reasons — a role, an interview, or
leveling up as a professional developer. Lessons should build toward the kind
of fluency an employer expects: building components, managing state, handling
events, fetching data, and reasoning about how React re-renders.

## Starting point (2026-07-13)
- Comfortable *reading* JavaScript, but shaky on the fundamentals:
  function syntax, array methods (`.map`, `.filter`), destructuring, async.
- New to React itself.

## Teaching stance
- Ground React in job-relevant skills, not toy demos.
- Shore up the specific JS gap *when it appears* in a React context, rather
  than front-loading a whole JS course.
- Favor the modern React employers use today: function components + hooks.

## Success looks like
Able to build a small interactive app from scratch (components, props, state,
events, a data fetch) and explain out loud why it re-renders when it does.
