Your First Component

← Index · Lesson 1 · The atom of every React app

Every React app — including the ones you'll build on the job — is made of components. A component is just a JavaScript function that returns some markup. That's the whole idea. Master this and everything else in React is a variation on it. [react.dev]

1. A component is a function that returns markup

function Greeting() {
  return <h1>Hello, React</h1>;
}

Two rules do most of the work here:

  • Capital letter. Greeting, not greeting. React uses the capital to tell your components apart from real HTML tags like <h1>.
  • It returns markup. That HTML-looking syntax is called JSX. It's not HTML and it's not a string — it's a JavaScript expression you're returning.
JS shore-up. That arrow-free function Greeting() { return ... } is a plain function declaration — same as any JS function, it just happens to return markup instead of a number. Nothing React-specific about the function keyword.

2. Components nest inside other components

You build a UI by putting components inside components, like Lego bricks. The top one is usually exported as the app's entry point.

function Greeting() {
  return <h1>Hello, React</h1>;
}

export default function App() {
  return (
    <div>
      <Greeting />
      <Greeting />
    </div>
  );
}

Notice a component is used like a self-closing tag: <Greeting />. And App returns a single <div> wrapping both greetings — because of the next rule.

3. Return exactly one root element

A component can only return one top-level element. If you need siblings, wrap them — in a <div>, or in an empty Fragment <>...</> when you don't want an extra div in the page.

return (
  <>
    <h1>Title</h1>
    <p>Body</p>
  </>
);

You can now read any React file and pick out its components, know why they're capitalized, and know why they wrap their output. That's the vocabulary the rest of the course is built on.
